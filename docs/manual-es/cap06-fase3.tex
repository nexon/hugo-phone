\chapter{Fase 3: DOSBox y Hugo}

\section{Objetivo}

Tener Hugo ejecutándose en la Pi, mostrado en pantalla por HDMI y jugable con teclado USB. Esta es la última pieza individual antes de unirlo todo en la Fase 4.

\section{Estado legal del juego}

Hugo es \emph{abandonware}: software comercial cuyo soporte por parte del editor original cesó hace décadas y cuya distribución pasiva es ampliamente tolerada aunque no formalmente autorizada. Sitios como \texttt{myabandonware.com} y \texttt{abandonwaregames.net} alojan copias descargables. La situación legal varía por jurisdicción, pero el uso doméstico y educativo no ha sido objeto de litigios.

\section{Paso 1: Conseguir Hugo}

La saga tiene varias entregas y los formatos varían:

\begin{itemize}
  \item \textbf{Hugo 1 / 2 / 3} (1990--1992) vienen como ZIP que al descomprimir deja un \texttt{HUGO.EXE} listo para ejecutar.
  \item \textbf{Hugo 4 y posteriores} suelen distribuirse como imagen ISO de CD-ROM con \texttt{INSTALL.EXE}, \texttt{INSTALLW.EXE} y \texttt{HUGO.EXE}. Requieren correr el instalador dentro de DOSBox la primera vez.
\end{itemize}

Una vez descargado, transferir el archivo a la Pi:

\begin{shellcode}
# Desde la Mac, ZIP (Hugo 1/2/3):
scp ~/Downloads/Hugo*.zip beto@hugo-pbx.local:/tmp/

# Desde la Mac, ISO (Hugo 4 y otros CD-ROM):
scp ~/Downloads/hugo4.iso \
  beto@hugo-pbx.local:/opt/hugo-phone/games/
\end{shellcode}

\subsection{Caso A: ZIP con \texttt{HUGO.EXE} suelto}

\begin{shellcode}
sudo mkdir -p /opt/hugo-phone/games/HUGO
sudo unzip /tmp/Hugo*.zip -d /opt/hugo-phone/games/HUGO
sudo chown -R beto:beto /opt/hugo-phone/games/HUGO
ls /opt/hugo-phone/games/HUGO/
\end{shellcode}

Debería listar \texttt{HUGO.EXE} (o un nombre similar). Pasar al Paso 4.

\subsection{Caso B: ISO de CD-ROM (Hugo 4)}

El ISO permanece como archivo. DOSBox lo monta como unidad \texttt{D:} con \texttt{imgmount}. La primera vez hay que correr el instalador (\texttt{INSTALL.EXE}, no \texttt{INSTALLW.EXE} que es para Windows) para que copie archivos a \texttt{C:\textbackslash{}HUGO\textbackslash{}} y configure el audio.

\section{Paso 2: Instalar DOSBox}

DOSBox (clásico) está en los repos oficiales de Raspberry Pi OS / Debian Trixie:

\begin{shellcode}
sudo apt update
sudo apt install -y dosbox xinit openbox xserver-xorg
which dosbox
\end{shellcode}

\begin{nota}
El paquete \texttt{dosbox-staging} no está disponible en el repositorio estándar de la Pi al momento de escribir este manual. Si en el futuro lo está, las opciones del \texttt{dosbox-hugo.conf} requerirían adaptarse a la sintaxis de \emph{Staging}.
\end{nota}

\section{Paso 3: Instalador del juego (sólo Caso B / ISO)}

Conectar HDMI + teclado USB a la Pi y loguearse en la TTY local (no por SSH). DOSBox necesita pantalla.

Lanzar el instalador con el ISO montado como \texttt{D:}:

\begin{shellcode}
startx /usr/bin/dosbox -- \
  -c "mount c /opt/hugo-phone/games" \
  -c "imgmount d /opt/hugo-phone/games/hugo4.iso -t iso" \
  -c "d:" \
  -c "INSTALL.EXE"
\end{shellcode}

Dentro del instalador:

\begin{itemize}
  \item \textbf{Destino}: \texttt{C:\textbackslash{}HUGO} (o lo que sugiera por defecto).
  \item \textbf{Sound card}: Sound Blaster 16, base \texttt{220}, IRQ \texttt{7}, DMA \texttt{1}. Coincide con la sección \texttt{[sblaster]} del \texttt{dosbox-hugo.conf}.
  \item Aceptar y salir del instalador.
\end{itemize}

Verificar que los archivos quedaron copiados:

\begin{shellcode}
ls /opt/hugo-phone/games/HUGO/
\end{shellcode}

\section{Paso 4: Verificar la configuración de DOSBox}

El script de instalación copió \texttt{dosbox-hugo.conf} a \texttt{/opt/hugo-phone/games/}. Verificar:

\begin{shellcode}
head -30 /opt/hugo-phone/games/dosbox-hugo.conf
\end{shellcode}

Si el ejecutable del juego no se llama exactamente \texttt{HUGO.EXE} o vive en una subcarpeta distinta, editar la sección \texttt{[autoexec]}:

\begin{shellcode}
nano /opt/hugo-phone/games/dosbox-hugo.conf
\end{shellcode}

Sección por defecto (Caso B, Hugo 4 ISO con CD presente):

\begin{plaincode}
[autoexec]
mount c /opt/hugo-phone/games
imgmount d /opt/hugo-phone/games/hugo4.iso -t iso
c:
cd HUGO
HUGO.EXE
exit
\end{plaincode}

Para Caso A (HUGO.EXE suelto, sin ISO), comentar la línea \texttt{imgmount}.

\section{Paso 5: Lanzar Hugo}

DOSBox del paquete de Debian usa SDL 1.2 y requiere un servidor X (no soporta KMSDRM directo). El camino más simple es \texttt{startx} + DOSBox dentro:

\begin{shellcode}
startx /usr/bin/dosbox -conf /opt/hugo-phone/games/dosbox-hugo.conf
\end{shellcode}

Esto inicia un X11 mínimo, ejecuta DOSBox dentro y, al cerrar DOSBox, X termina y vuelve a la TTY.

\begin{nota}
\texttt{startx} sólo funciona desde una TTY local (no por SSH). Para pruebas remotas se puede usar SSH con X forwarding (\texttt{ssh -Y}), pero el rendimiento es muy inferior y no se recomienda para jugabilidad real.
\end{nota}

\section{Paso 6: Arranque automático del juego}

Configurar la Pi para que al encender entre directamente a Hugo, sin intervención del usuario.

\subsection{Habilitar autologin en TTY1}

\begin{shellcode}
sudo raspi-config
\end{shellcode}

Navegar a: \textsf{System Options} $\to$ \textsf{Boot / Auto Login} $\to$ \textsf{Console Autologin}. Guardar y salir.

\subsection{Ejecutar DOSBox al iniciar sesión}

Editar el \texttt{.bash\_profile} del usuario \texttt{beto}:

\begin{shellcode}
nano ~/.bash_profile
\end{shellcode}

Agregar al final:

\begin{plaincode}
if [ -z "$DISPLAY" ] && [ "$(tty)" = "/dev/tty1" ]; then
  startx /usr/bin/dosbox \
    -conf /opt/hugo-phone/games/dosbox-hugo.conf
fi
\end{plaincode}

La condición evita que DOSBox se ejecute si se ingresa al sistema por SSH o desde una TTY distinta de la 1.

Al reiniciar, la secuencia debe ser:

\begin{enumerate}
  \item Boot del kernel
  \item Autologin como \texttt{beto} en \texttt{tty1}
  \item \texttt{startx} arranca X11
  \item DOSBox arranca, ejecuta \texttt{[autoexec]} y aparece Hugo
\end{enumerate}

\section{Paso 7: Prueba de jugabilidad}

Con un teclado USB conectado a la Pi y Hugo cargado, probar:

\begin{itemize}
  \item Flechas direccionales $\to$ movimiento del personaje
  \item Barra espaciadora $\to$ acción
  \item \texttt{Esc} $\to$ menú/salir
\end{itemize}

Si Hugo se siente excesivamente rápido o lento, ajustar la línea \texttt{cycles} en \texttt{dosbox-hugo.conf}:

\begin{tomlcode}
[cpu]
cycles = fixed 8000   # Aumentar si va lento, disminuir si va rapido
\end{tomlcode}

Hugo original requería aproximadamente un 386SX a 25\,MHz. Empezar con \texttt{cycles = fixed 8000} y calibrar.

\section{Paso 8: Salir limpiamente}

Dos pulsaciones de \texttt{Esc} salen al menú de Hugo. La opción \emph{Quit} regresa al prompt de DOSBox; el comando \texttt{exit} del bloque \texttt{[autoexec]} cierra DOSBox automáticamente.

Para salir de emergencia (por ejemplo si el juego se cuelga): \texttt{Ctrl}+\texttt{F9} es el \emph{kill switch} de DOSBox.

\section{Resolución de problemas}

\subsection{\texttt{startx} no encuentra display o falla}

Confirmar que están los paquetes del servidor X:

\begin{shellcode}
sudo apt install -y xinit openbox xserver-xorg
\end{shellcode}

Y que estás logueado en una TTY local (no SSH). Verificar con \texttt{tty} (debe responder \texttt{/dev/tty1} o similar, no \texttt{/dev/pts/*}).

\subsection{Hugo se cuelga en la intro o las cinemáticas}

DOSBox en arquitectura ARM ocasionalmente tiene problemas con la decodificación de animaciones VGA. Pulsar cualquier tecla para saltar la intro suele resolverlo. Si persiste, bajar la calidad del \emph{output}:

\begin{tomlcode}
[sdl]
output = surface
\end{tomlcode}

\subsection{El audio se entrecorta}

Aumentar el buffer del mezclador:

\begin{tomlcode}
[mixer]
prebuffer = 50
blocksize = 2048
\end{tomlcode}

Si aún hay cortes, bajar la calidad: \texttt{rate = 22050}.

\subsection{El juego pide CD pero el ISO está montado}

Verificar que el \texttt{imgmount} no falló silenciosamente:

\begin{shellcode}
ls -la /opt/hugo-phone/games/hugo4.iso
\end{shellcode}

Debe existir y ser legible por el usuario \texttt{beto}. Si no, ajustar permisos:

\begin{shellcode}
sudo chown beto:beto /opt/hugo-phone/games/hugo4.iso
\end{shellcode}

\subsection{Hugo arranca pero no responde al teclado}

\begin{itemize}
  \item Verificar que \texttt{core = dynamic} esté presente en la sección \texttt{[cpu]}.
  \item Algunos títulos requieren \texttt{cputype = 386} en lugar de \texttt{auto}.
  \item Asegurarse de que la ventana de DOSBox tiene el foco (con \texttt{autolock = true} se captura el teclado al hacer clic).
\end{itemize}
